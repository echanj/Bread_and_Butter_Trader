Overview

Our algorithmic trading journey has led to the humble development of a fully automated backtesting and screening engine, designed around concepts such as generalized pairs trading, cross-sectional analysis, PCA, factor modeling, mean reversion, OU process, Kelly factor and trading parameter optimization (to name a few). Many concepts are wrapped into the engine: A standalone framework which functions as a Trading Automaton (or meta-simulator). We are constantly integrating and testing new methods and approaches in our platform. This is done via running many trading experiments over historical data, typically termed backtesting simulation. this amounts to alot of data which we can then use to optimise the overall system perfromace. This is done using a combination of well accepted machine learning techniques as well as cutting edge AI research.

The type of micro-strategies the software incorporates are currently based on insights from Dr. Ernest Chan and the well-regarded paper, "Statistical Arbitrage in the U.S. Equities Market" by Marco Avellaneda and Jeong-Hyun Lee. One approach is to simulate trading using multiple historical time series data blocks and ranks profitable, low-risk Z-score signals using metrics such as the Sharpe ratio. Given its current lower-frequency operation, trading with full automation is not required, allowing for strategic discretion in the trading process.

The workflow still supports discretionary order submissions (i.e. clients do thier own analysis and submit orders manually) and leverages the strength of mean-reverting Z-score signals and their deviation from the mean. To enhance the automaton's suggestions, additional technical analysis is incorporated, utilizing tools like ARIMA, GARCH, and MSM. While these metrics can be further analyzed using large language models (LLMs) for additional insight, we find this to be less essential, especially for automated execution. Most technical analysis outputs are set to default when running our system. This site only shows the default outputs as examples. However, we do tailor outputs (adjust parameters, etc) for our clients upon request.

Example Manual Use Case

The time series data block was updated on the desired trade date, here we select an instance of Single-instrument Trader on S&P 500 data. (View output). Ignoring the table with quantitative metrics, we quickly scan the available Z-score signals. Notably, the DXCM Z-score sits just below the optimized entry condition.

Just above the output table are links to supporting analysis. Clicking the link for DXCM provides more insights: (DXCM stock analysis).

The Z-score itself doesn‚Äôt provide relative prices, but these are available in the first chart of the supporting analysis (shown below).

Suggested entry for a long position is $69.21, with an exit at $69.51. These values are optimized to minimize risk in automated trading. Deciding the entry is reasonable, before the market opens, we place a STOP LIMIT order through our trading platform. Assuming sufficient momentum, we add a take-profit order, setting it near the short exit target (around $70.10). After next day market close, as shown in the chart below, the trade was successful. However, we may have set the take profit to have been above the suggested short entry threshold. Interestingly, the price reversal began near the suggested short entry. NB: If we had stuck to the suggested long exit, we would have mitigated a lot of risk.
